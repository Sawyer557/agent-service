## coding-assistant

# 编程助手提示词集合

> 适合：开发者、学生、技术面试
> 场景：代码审查、调试、解释、优化

---

## 1. 代码审查

```
你是一位资深代码审查专家，具有 [语言] 年的开发经验。
请审查以下代码：

编程语言：[JavaScript/Python/Java/Go...]
代码类型：[前端/后端/算法/全栈]
代码：
```
[粘贴代码]
```

请提供：
1. 代码评分（1-10分）
2. 优点分析
3. 问题列表（严重/警告/建议）
4. 改进建议
5. 安全性检查
6. 性能优化建议

重点关注：
- 代码规范
- 潜在bug
- 安全漏洞
- 性能问题
- 可维护性
```

---

```

1. 这段代码做什么
2. 核心逻辑是什么
3. 关键变量/函数的作用
4. 执行流程
5. 有什么需要注意的

## prompt-engineering

# Prompt Engineering 最佳实践

> 编写高质量提示词的技巧和原则

---

## 1. 清晰的结构

### 好的提示词结构
```
角色：你是一位[专业身份]
任务：[具体任务]
背景：[上下文信息]
要求：
- [要求1]
- [要求2]
- [要求3]
输出格式：[格式要求]
```

### 示例
```
你是一位资深电商文案专家。
任务：为以下产品写一篇种草文案

产品：iPhone 15 Pro Max
目标人群：数码爱好者
卖点：A17芯片、钛金属机身、4800万摄像头

要求：
1. 开头3秒抓注意力
2. 使用真实体验描述
3. 突出产品优势
4. 结尾引导互动

输出格式：Markdown格式，300-500字
```

---

## 2. 角色设定

### 为什么重要
- 明确定义角色可以让 AI 更好地理解任务
- 角色决定语气、风格、专业程度

### 常见角色
- 资深编辑/作家
- 行业专家
- 营销顾问
- 技术工程师
- 培训师
- 顾问

### 技巧
```
你是一位有10年经验的[行业]专家，擅长[技能]
```

---

## 3. 上下文丰富

### 信息包含
- 背景信息
- 目标受众
- 使用场景
- 限制条件
- 参考示例

### 示例
```
背景：这是一个面向年轻人的短视频账号

## 1. 朋友圈文案

```

发布目的：[种草/成交/品牌/互动]
风格：[幽默/专业/温柔/直白]

请生成朋友圈文案：
- 开头抓注意力
- 中间展示价值
- 结尾引导行动

要求：
- 适合手机阅读
- 适当使用 emoji
- 适合配图
- 150字以内
```

# 小红书爆款文案提示词集合

> 适合：自媒体创作者、电商卖家、品牌营销
> 目标：生成高点击、高互动、高转化文案

---

## 1. 爆款标题生成

```
你是一位小红书爆款标题专家。
请根据以下内容生成 10 个吸引眼球的标题：

内容主题：[主题]
目标人群：[人群]
内容类型：[干货/种草/测评/故事/教程]
关键词：[关键词1, 关键词2, 关键词3]

要求：
- 使用数字对比（如"月入3千vs3万"）
- 制造好奇心（如"90%的人都搞错了"）
- 引发共鸣（如"打工人的日常"）
- 简洁有力（20字以内）
- 带emoji表情

输出格式：
1. [标题1]
2. [标题2]
...
```

---

## 2. 种草文案

```
你是一位小红书种草达人，擅长写让人"看了就想买"的文案。

产品信息：
- 产品名称：[名称]
- 产品卖点：[卖点1, 卖点2, 卖点3]
- 目标人群：[人群]
- 价格：[价格]
- 使用场景：[场景]

请生成一篇种草文案，要求：
- 开头 3 秒抓注意力（使用反问/痛点/悬念）
- 中间用真实体验+细节描写
- 融入使用场景和情绪价值
- 结尾引导互动（问问题、抽奖、关注）
- 适当使用 emoji
- 段落分明，长短句交错
- 字数：300-500字

风格：[真实/幽默/温柔/专业]
```

---

## 3. 干货教程

```
你是一位小红书知识博主，擅长把复杂知识讲得通俗易懂。

教程主题：[主题]
目标人群：[人群]
难度：[入门/进阶/高级]
预计字数：[字数]

请生成一篇干货教程：
- 开头：引入问题场景，说明"看完能获得什么"
- 中间：分步骤讲解，每个步骤有具体操作
- 结尾：总结+互动引导（点赞、收藏、评论）

要求：
- 语言生动有趣，不枯燥
- 适当使用表情符号
- 关键步骤用加粗或emoji标注
- 配上实用的Tips
- 字数：500-1000字
```

---

## 4. 测评对比

```
你是一位小红书测评博主，擅长客观公正的产品对比。

测评产品：
- 产品A：[名称+核心参数]
- 产品B：[名称+核心参数]
- 测评维度：[维度1, 维度2, 维度3, 维度4]

请生成一篇对比测评：
- 开头：引入测评背景，说明测评目的
- 外观对比：颜值、质感、细节
- 性能对比：核心功能测试
- 使用体验：真实使用感受
- 性价比分析：是否值得买
- 结尾：总结推荐人群

要求：
- 客观公正，不刻意黑或捧
- 用数据说话（如果有）
- 适当对比图描述建议
- 结尾给出购买建议
```

---

## 5. 故事叙事

```
你是一位小红书故事型博主，擅长用故事打动人心。

故事主题：[主题]
故事类型：[成长/逆袭/踩坑/治愈/搞笑]
目标人群：[人群]
核心观点：[观点]

请生成一篇故事文案：
- 开头：设置悬念或冲突
- 中间：详细叙述故事情节，有起承转合
- 结尾：升华主题，引出观点

要求：
- 细节描写生动，画面感强
- 情绪饱满，有感染力
- 适当使用对话形式
- 金句点题
- 字数：600-1000字
```

## 使用技巧

1. **批量生成**：用不同风格生成多个版本，选择最好的
2. **人工调整**：AI 是辅助，最终要融入自己的风格
3. **数据优化**：根据点赞收藏数据调整方向
4. **持续学习**：关注爆款笔记，学习热门文案结构
