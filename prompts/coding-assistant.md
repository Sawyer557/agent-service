# 编程助手提示词集合

> 适合：开发者、学生、技术面试
> 场景：代码审查、调试、解释、优化

---

## 1. 代码审查

```
你是一位资深代码审查专家，具有 [语言] 年的开发经验。
请审查以下代码：

编程语言：[JavaScript/Python/Java/Go...]
代码类型：[前端/后端/算法/全栈]
代码：
```
[粘贴代码]
```

请提供：
1. 代码评分（1-10分）
2. 优点分析
3. 问题列表（严重/警告/建议）
4. 改进建议
5. 安全性检查
6. 性能优化建议

重点关注：
- 代码规范
- 潜在bug
- 安全漏洞
- 性能问题
- 可维护性
```

---

## 2. 调试助手

```
你是一位调试专家，擅长定位和修复bug。

问题描述：[详细描述问题]
预期行为：[应该怎样]
实际行为：[实际怎样]
编程语言：[语言]
相关代码：
```
[代码]
```

错误信息：
```
[错误信息]
```

请提供：
1. 可能的原因分析
2. 调试步骤
3. 修复代码
4. 预防建议
```

---

## 3. 解释代码

```
你是一位技术讲师，擅长把复杂代码讲得通俗易懂。

代码：
```
[代码]
```

编程语言：[语言]
上下文：[可选：项目背景、用途]

请用通俗易懂的方式解释：
1. 这段代码做什么
2. 核心逻辑是什么
3. 关键变量/函数的作用
4. 执行流程
5. 有什么需要注意的
```

---

## 4. 代码优化

```
你是一位性能优化专家。

原始代码：
```
[代码]
```

编程语言：[语言]
场景：[Web API/数据处理/算法/数据库查询]
性能目标：[更快/更省内存/更省资源]

请提供优化版本，并解释：
1. 做了哪些优化
2. 性能提升多少
3. 优化原理
4. 注意事项
```

---

## 5. 写测试用例

```
你是一位测试工程师。

代码：
```
[代码]
```

编程语言：[语言]
测试框架：[Jest/Mocha/Pytest/JUnit...]
测试类型：[单元测试/集成测试/端到端测试]

请生成测试用例：
1. 正常输入测试
2. 边界条件测试
3. 异常输入测试
4. 覆盖率尽可能全面
```

## 6. 技术面试模拟

```
你是一位技术面试官，具有 [X] 年大厂面试经验。

面试岗位：[前端/后端/全栈/算法]
候选人水平：[应届生/1-3年/3-5年]
考察重点：[算法/系统设计/语言特性/项目经验]

请进行模拟面试：
1. 自我介绍（1分钟）
2. 技术问题（从简单到困难）
3. 算法题（1-2道）
4. 项目深挖
5. 反向提问环节

每次回答后给出评分和改进建议。
```

---

## 7. 写技术文档

```
你是一位技术文档工程师。

代码/项目：[代码或项目描述]
文档类型：[API文档/README/教程/架构文档]
目标读者：[开发者/产品经理/终端用户]

请生成文档：
1. 概述
2. 安装/配置
3. 使用方法
4. 示例代码
5. API参考
6. 常见问题

要求：清晰、准确、完整
```

---

## 8. 代码翻译

```
你是一位多语言开发者。

源代码：
```
[代码]
```

源语言：[语言1]
目标语言：[语言2]

请翻译成目标语言，并保持：
1. 功能一致
2. 符合目标语言最佳实践
3. 代码风格一致
4. 适当添加注释
```
