# Prompt Engineering 最佳实践

> 编写高质量提示词的技巧和原则

---

## 1. 清晰的结构

### 好的提示词结构
```
角色：你是一位[专业身份]
任务：[具体任务]
背景：[上下文信息]
要求：
- [要求1]
- [要求2]
- [要求3]
输出格式：[格式要求]
```

### 示例
```
你是一位资深电商文案专家。
任务：为以下产品写一篇种草文案

产品：iPhone 15 Pro Max
目标人群：数码爱好者
卖点：A17芯片、钛金属机身、4800万摄像头

要求：
1. 开头3秒抓注意力
2. 使用真实体验描述
3. 突出产品优势
4. 结尾引导互动

输出格式：Markdown格式，300-500字
```

---

## 2. 角色设定

### 为什么重要
- 明确定义角色可以让 AI 更好地理解任务
- 角色决定语气、风格、专业程度

### 常见角色
- 资深编辑/作家
- 行业专家
- 营销顾问
- 技术工程师
- 培训师
- 顾问

### 技巧
```
你是一位有10年经验的[行业]专家，擅长[技能]
```

---

## 3. 上下文丰富

### 信息包含
- 背景信息
- 目标受众
- 使用场景
- 限制条件
- 参考示例

### 示例
```
背景：这是一个面向年轻人的短视频账号
受众：18-25岁
场景：种草推荐
限制：不涉及政治敏感话题
参考风格：@李佳琦
```

---

## 4. 具体化要求

### ❌ 模糊
```
写一篇关于AI的文章
```

### ✅ 具体
```
写一篇1500字的文章，主题是"AI如何改变电商运营"
- 包含3个具体案例
- 使用通俗易懂的语言
- 适合非技术背景读者
- 结尾给出行动建议
```

---

## 5. 格式化输出

### 指定格式
- Markdown
- JSON
- 列表
- 表格
- 代码块

### 示例
```
输出格式要求：
1. 使用Markdown
2. 二级标题分隔章节
3. 关键点用加粗
4. 列表用-符号
```

---

## 6. 迭代优化

### 初始版本
```
写一篇关于健身的文章
```

### 优化版本1
```
你是一位健身教练。写一篇关于"新手如何开始健身"的文章，500字，针对25-35岁上班族。

要求：
- 给出3个具体建议
- 包含常见误区
- 语言通俗易懂
- 适合健身小白
```

### 优化版本2
```
角色：资深健身教练，10年教学经验
任务：写一篇健身入门指南

目标读者：25-35岁上班族，之前没有健身经验

文章要求：
- 500字左右
- 包含：
  1. 入门前的准备（体检、装备）
  2. 3个最适合初学者的动作（附要点）
  3. 常见5个误区
  4. 第一周训练计划
- 语气：专业但亲切
- 结尾：鼓励性话语

输出：Markdown格式
```

## 7. Few-shot 学习

### 什么是 Few-shot
给 AI 展示几个示例，让它理解你想要的输出格式

### 示例
```
任务：生成产品标题

示例：
输入：无线蓝牙耳机
输出：🎧 真无线蓝牙耳机 | 降噪+长续航 | 兼容苹果安卓

输入：iPhone15手机壳
输出：
```

## 8. 链式思考 (Chain of Thought)

### 技巧
让 AI 先思考再回答

### 示例
```
在回答之前，请先思考：
1. 这个问题的核心是什么？
2. 受众最关心什么？
3. 最好的解决方案是什么？

然后给出你的回答。
```

---

## 9. 检查和约束

### 添加约束
```
- 限制字数在300字以内
- 不要使用专业术语
- 避免提到具体品牌
- 保持积极正面的语气
```

### 添加检查点
```
请在完成前检查：
1. 是否解决了用户的问题？
2. 语言是否通顺？
3. 格式是否正确？
```

## 10. 常见模式

### 问答模式
```
Q: [问题]
A: [回答]

请按照以上格式回答以下问题：
Q: [问题1]
A:
```

### 教学模式
```
你是一位[老师]。请用[学生能理解]的方式讲解[概念]。

要求：
- 从简单到复杂
- 包含例子
- 最后有练习题
```

### 角色扮演
```
你是一位[角色]。请根据以下情境做出反应：

情境：[场景描述]
你的反应：
```

## 提示词模板库

### 面试辅助
```
你是一位[年限]年经验的[行业]面试官。
请针对以下职位进行模拟面试：

职位：[职位]
级别：[级别]

请按以下流程进行：
1. [环节1]
2. [环节2]
3. [环节3]

每次回答后给出评分和改进建议。
```

### 内容创作
```
你是一位[平台]内容专家。
请为以下[类型]内容生成[具体]：

主题：[主题]
目标：[目标]
风格：[风格]

要求：
1. [要求1]
2. [要求2]
3. [要求3]
```

### 代码辅助
```
你是一位[语言]编程专家。
请帮我[任务]：

[代码上下文或需求]

要求：
- [约束1]
- [约束2]
- [性能要求]
```

---

## 常见错误

1. **角色不明确** → 添加具体角色设定
2. **要求太模糊** → 数字化、具体化
3. **缺少上下文** → 添加背景和受众信息
4. **格式不指定** → 明确输出格式
5. **没有示例** → 添加 Few-shot 示例
